# Doodle tab — replace the "decorate" tab with a daily doodle prompt

**Date:** 2026-06-27
**Scope:** `index.html`, `styles.css`, `js/main.js`, new `js/prompts.js`, and a
rename/rewrite of `js/customDecorate.js` → `js/doodleDecorate.js`. No backend
(`scripts/`) or `data/messages.json` contract changes.

## Summary

The second tab is reframed. Today "decorate" lets the user draw on a full-stage
canvas and drag the daily note around. The new **doodle** tab instead gives the
user a **daily one-word drawing prompt** (a random object, e.g. *cat*) shown at
the top, with a **bounded portrait canvas** below it to draw that object in. The
daily note/quote is not shown in doodle mode. Auto mode is unchanged.

## Goals

1. **Daily doodle prompt.** One random single-word object per *day*, generated
   client-side and seeded deterministically by date (`YYYY-MM-DD`). Same word for
   everyone on a given day; AM and PM share the same prompt. Works offline.
2. **Bounded canvas.** A centered rounded vertical (portrait) rectangle. All
   drawing is confined to it — strokes are clipped to the rounded-rect.
3. **Remove the move tool.** There is no draggable note in doodle mode, so the
   `move` tool is gone. Remaining tools: pencil / eraser / undo / clear / save.
4. **Save names the prompt.** Saved PNG downloads as `mindbob_<prompt>_<date>.png`
   (am/pm dropped). The PNG contains the **canvas drawing only** (on the palette
   background) — no prompt text baked into the image.
5. **Persist per day.** A doodle is saved/restored per date, not per note id.

## Non-goals

- No change to the auto-mode note display, the cron, the LLM pipeline, or the
  `data/messages.json` contract.
- The prompt is NOT generated by the backend and is NOT stored in any data file.
- The PNG does not include the prompt word as rendered text (filename only).

## Prompt generation (`js/prompts.js`, new)

A new single-purpose module mirroring the seeding pattern of `palette.js` /
`doodles.js`.

- A curated, bundled list of ~60–80 simple, drawable single-word nouns
  (e.g. cat, house, tree, cup, boat, star, fish, leaf, key, bird, …).
- `promptFor(seed)` → picks one word deterministically using `hashString` +
  `seededRng` / `pick` from `util.js`, exactly like the existing visual seeding.
- The **seed is the date string** (`entry.date`, `YYYY-MM-DD`), not `entry.id`,
  so AM and PM yield the same prompt.
- Export shape: `promptFor(dateSeed) -> string`.

Determinism rule (matches repo convention): no `Math.random()`; seed from the
date. Same day → same word.

## UI (`index.html` + `styles.css`)

- **Toggle label** `decorate` → `doodle`. Rename the button id `modeCustom` →
  `modeDoodle` (and `modeAuto` stays) for clarity; update `main.js` refs.
- **Prompt element** at the top of the stage, e.g.
  `<div id="doodlePrompt" class="doodle-prompt">draw: <strong>cat</strong></div>`,
  shown only in doodle mode (CSS via `.app[data-mode="doodle"]`).
- **Mode value** `data-mode` becomes `"auto"` / `"doodle"` (was `"custom"`).
  Update the existing CSS selectors that key off `data-mode="custom"` (e.g.
  `.app[data-mode="auto"] .layer-canvas { display:none }` and friends).
- **Canvas** is restyled to a centered portrait rounded rectangle: constrained
  `max-width`, taller-than-wide aspect, `border-radius`, sitting below the
  prompt. The `#messageEl` note layer is hidden in doodle mode.
- **Toolbar:** remove the `move` button entirely. Tools left: pencil / eraser,
  swatches (seeded pencil palette), undo / clear / save. The `.toolbar[hidden]`
  `display:none` trap still applies.

## `js/doodleDecorate.js` (rename + rewrite of `customDecorate.js`)

Lazy-imported by `main.js` only when entering doodle mode (same as today).

Removed vs. current `customDecorate.js`:
- All move/drag logic: `onMsgDown/Move/Up`, `applyMsgTransform`, `msgX`/`msgY`,
  `dragStart`, the `move` button wiring, and `is-draggable`/`is-dragging`.
- The note is no longer composited or positioned here.

Changed:
- **Undo snapshots** become canvas-pixels-only (drop `msgX`/`msgY`).
- **`fitCanvas`** sizes to the portrait canvas rectangle (its own
  `getBoundingClientRect()`), not the whole stage. After (re)sizing and setting
  the DPR transform, apply a **rounded-rect clip** to the context so strokes stay
  inside the rounded canvas. (DPR-aware sizing and CSS-pixel stroke storage are
  preserved.)
- **Default tool** is `pencil` (no move).
- **Pointer math** (`pos`) keys off the canvas rect, which now equals the canvas
  element — already correct.

Persistence:
- Key by **date**: `mindbob:doodle:<entry.date>`. New key prefix
  (`mindbob:doodle:`) so the old `mindbob:decoration:*` keys are simply abandoned
  (and pruned: on persist, remove any other `mindbob:doodle:*` key). One doodle
  per day; it resets when the date rolls over.
- Payload is canvas pixels only: `{ img: dataURL }` (no `msgX/msgY`). All storage
  access stays `try/catch`-wrapped.
- `restore()` still runs once per load in `activate()` after `fitCanvas()`,
  redrawing the saved bitmap scaled to the live canvas.

Save:
- **Filename:** `filename()` → `mindbob_<prompt>_<date>.png`, dropping empty
  parts. `<prompt>` is the day's word; `<date>` is `entry.date`. No slot, no
  theme name.
- **Content:** export = palette background + canvas drawing. Drop the
  `drawMessageText` / `wrapText` text-baking entirely (PNG has no prompt text).

## Palette

The palette mechanism is unchanged. Doodle mode uses the current entry's
`state.palette` for the canvas background and pencil swatches (already loaded in
`main.js`). Known minor cosmetic effect: the palette is seeded per note (AM≠PM)
while a doodle persists per day, so the background can shift AM→PM while saved
strokes keep their colors. Accepted as-is; not restructuring palettes.

## Module map delta

| `js/doodleDecorate.js` | **Renamed/rewritten** from `customDecorate.js`. Drawing-only, bounded canvas, per-day persistence. `createDoodleDecorator(refs, state)`. |
| `js/main.js` | Mode `custom`→`doodle`; lazy-import `doodleDecorate.js`; load `promptFor(entry.date)`, render prompt; updated refs/ids. |

## Invariants preserved / updated

- Deterministic-per-seed visuals: the prompt joins palette/doodles as a
  date-seeded, no-`Math.random()` value.
- DPR-aware canvas, CSS-pixel stroke storage, snapshot-based undo, per-key
  storage pruning, `try/catch` storage degradation, `.toolbar[hidden]` rule —
  all preserved.
- The "note can never be erased" invariant becomes moot in doodle mode (no note
  layer present); the canvas is the only drawable surface.
